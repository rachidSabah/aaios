# AAiOS Administrator Guide

This document defines administrative procedures, security configurations, and capacity planning guidelines for managing an enterprise Agentic AI Operating System (AAIOS) cluster.

---

## 1. Upgrades and Release Channel Management

AAiOS supports the following release channels for updates, configured using the `--channel` parameter on the update CLI:

*   **Stable**: Production-ready, fully tested releases. Recommended for standard workloads.
*   **LTS (Long Term Support)**: Enterprise-grade builds focusing on security patches and maximum stability.
*   **Beta**: Release candidates undergoing final validation checks.
*   **Nightly**: Automated developer builds directly compiling master branch commits. Unstable.
*   **Enterprise**: Custom patches tailored to specific private network constraints.

To pin a specific version and block updates, use the CLI:
```bash
aaios update --pin 5.3.2
```

---

## 2. Backup Retention Policy

Backups can consume substantial disk space over time. Admins should enforce the following retention schedule:

*   **Daily Backups**: Retain for 7 days.
*   **Weekly Backups**: Retain for 4 weeks.
*   **Monthly Backups**: Retain for 12 months.
*   **Immutable Snapshots**: Retain indefinitely for audit compliance or until retired by release management.

To purge older backups:
```bash
aaios backup delete <BACKUP_ID>
```

---

## 3. Cryptographic and Security Settings

### Access Permissions
To prevent unauthorized access to API keys, restrict access to the `secrets/` directory:
*   **Windows**: Set NTFS permissions to only allow the service account full access.
*   **Linux/WSL**: Ensure folder permissions are set to `700` (`rwx------`). The self-healing engine automatically corrects this on unix hosts.

### SSL Certificates
Subagents communicating over internal node networks use SSL certificates. The system automatically creates self-signed certificates in `certificates/` during bootstrapping. To apply custom corporate-signed certs, replace `certificates/server.crt` and `certificates/server.key` with PEM-formatted certificates.

---

## 4. Capacity Planning

AAiOS stores metadata, experiences, and audit trails in SQLite databases. Keep track of storage growth using the following estimates:

*   **Audit logs (`database/audit.db`)**: ~500 bytes per event.
*   **Experiences (`database/experience.db`)**: ~2 KB per agent execution.
*   **Memory Indexes (`database/memory.db`)**: Depends on prompt sizes, averages ~5 KB per record.

Ensure the host disk has sufficient capacity to handle the projected daily write logs, and monitor storage alerts generated by `aaios monitor`.
